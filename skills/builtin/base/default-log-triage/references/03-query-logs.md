# 日志查询子技能

## 目标

从真实日志系统收集错误窗口、trace 相关日志和兜底模板证据，判断故障是否可在日志层面定位。

## 触发时机

- 服务目录和日志源解析完成后。
- 有 trace id 时优先 trace 查询。
- 没有 trace id、trace 查询为空、或症状像重复错误模式时执行模板查询。

## 工具

- `log.query_by_trace_id`
- `log.query_by_template`

## 查询策略

- `log.query_by_trace_id` 使用归整后的 trace id 和服务名。
- `log.query_by_template` 使用默认错误窗口模板和服务名。
- 优先保留异常类、错误码、请求路径、上下游服务、模板 id、堆栈帧、耗时和重试线索。
- 日志工具未配置、返回为空或被截断时，要在证据中明确标注。

## 输出要求

- 查询 key。
- 命中日志条数。
- 关键日志摘要。
- 是否截断。
- 工具配置或权限问题。
