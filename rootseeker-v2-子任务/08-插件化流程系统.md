# T8 插件化流程系统

## 1. 任务目标

本任务用于将 OpenClaw 插件系统思想融入 RootSeeker V2，把排查流程、工具、通道、连接器和策略都做成可声明、可发现、可注册的插件。

参考文档：

- `/Users/beisen/PycharmProjects/openclaw-analysis/04-插件和Provider系统分析.md`

---

## 2. 范围

本任务覆盖：

- 插件 manifest
- 插件发现
- 插件来源优先级
- 插件能力契约
- 插件注册 API
- 流程插件执行入口

本任务不覆盖：

- 插件市场
- 第三方插件签名分发
- 完整插件 UI

---

## 3. 输入

- 插件目录
- 插件 manifest
- 系统配置
- 运行时上下文

---

## 4. 输出

- `PluginManifestRecord`
- `PluginRegistry`
- `PluginRuntime`
- `RegisteredCapability`

---

## 5. 一级拆解

### `T8.1` 插件 manifest 规范

定义 RootSeeker 的插件声明文件。

### `T8.2` 插件发现与加载

支持多来源插件发现。

### `T8.3` 插件能力契约

声明插件能注册哪些能力。

### `T8.4` 流程插件

将排查流程插件化执行。

---

## 7. 风险

- 如果插件只支持工具，不支持流程，排查流程仍会硬编码
- 如果没有来源优先级，插件覆盖关系会混乱
- 如果没有能力契约，插件注册会变成动态黑盒

---

## 8. 验收标准

- 系统可以发现多个来源的插件
- 插件 manifest 能被解析和校验
- 插件能注册 flow、tool、channel、policy 等能力
- 一个排查流程可以通过插件形式被执行

---

## 9. 推荐实施顺序

1. 定义 manifest
2. 实现 discovery
3. 实现 manifest registry
4. 实现 plugin API
5. 实现 flow plugin
