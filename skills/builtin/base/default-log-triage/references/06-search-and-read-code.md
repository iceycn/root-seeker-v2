# 代码定位子技能

## 目标

用日志、trace 或告警文本中的可靠线索搜索代码，并读取最相关的代码片段作为证据。

## 触发时机

- 已经完成日志、trace、索引和仓库上下文检查。
- 存在异常类、方法名、文件路径、API 路由、错误码、队列/topic、SQL key、日志模板等线索。

## 工具

- `code.search`
- `code.read`

## 搜索准则

- 使用当前最具体的线索搜索，避免无信号的宽泛搜索。
- 优先使用日志和 trace 中出现的异常、方法、路径、模板。
- 如果 `code.search` 返回文件命中，优先读取第一批高相关命中。
- 如果症状包含 `file:line` 或 metadata 提供 `code_path`，可以直接读取该文件。
- 没有明确目标时不要随机读文件。

## 输出要求

- 搜索 query。
- 命中文件、行号、摘要。
- 读取文件路径和片段。
- 无法读取时的原因，例如索引未配置、仓库未注册、没有明确目标。
