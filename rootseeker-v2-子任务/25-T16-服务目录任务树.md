# T16 服务目录任务树

## 1. 目标

本任务把 `19-服务目录与日志源映射.md` 落成可执行任务树，确保服务目录只能通过 `Skill + 内置服务目录插件 + MCP` 使用。

## 2. 模块拆分

| 模块 | 目标 | 建议文件 |
|---|---|---|
| `service-catalog-contract` | 定义 `ServiceCatalogEntry` | `contracts/service_catalog.py` |
| `catalog-store` | 存储服务目录 | `service_catalog/store.py` |
| `catalog-loader` | 加载配置和版本 | `service_catalog/loader.py` |
| `catalog-plugin` | 内置服务目录插件 | `plugins/builtin/service_catalog/plugin.yaml` |
| `catalog-mcp-tools` | 注册 MCP 工具 | `mcp_servers/internal/catalog_tools.py` |
| `service-resolver` | 解析服务名 | `service_catalog/resolver.py` |
| `source-resolver` | 解析日志源/仓库/trace | `service_catalog/source_resolver.py` |
| `catalog-audit` | 审计目录查询 | `service_catalog/audit.py` |

## 3. 强制调用链

```text
Skill Step
  -> builtin.service-catalog plugin
  -> MCP catalog.resolve_service
  -> ServiceCatalogEntry
  -> downstream log/code/trace skills
```

## 4. 禁止事项

- 禁止默认 Flow 直接读取服务目录存储。
- 禁止 Agent Runtime 直接读取服务目录。
- 禁止未声明插件的服务映射进入生产链路。

## 5. 验收标准

- `tenant + environment + service_name` 能解析到日志源、仓库、索引和团队。
- 服务目录查询能追溯到 Skill 和 MCP 调用。
- 多租户、多环境不会串数据。
- 服务目录变更可审计、可版本化。
