# T15 日志数据面任务树

## 1. 目标

本任务把 `18-日志数据面契约.md` 落成可执行任务树，确保日志查询必须通过 `Skill + 内置日志插件 + MCP` 完成。

## 2. 模块拆分


| 模块                        | 目标                    | 建议文件                                    |
| ------------------------- | --------------------- | --------------------------------------- |
| `log-source-contract`     | 定义 `LogSource`        | `contracts/log_source.py`               |
| `query-template-contract` | 定义 `LogQueryTemplate` | `contracts/log_query.py`                |
| `log-plugin`              | 内置日志插件声明              | `plugins/builtin/log_query/plugin.yaml` |
| `mcp-log-tools`           | 注册日志 MCP 工具           | `mcp_servers/internal/log_tools.py`     |
| `query-renderer`          | 渲染 SQL 模板             | `log_data/query_renderer.py`            |
| `time-window-policy`      | 计算时间窗                 | `log_data/time_window.py`               |
| `trace-id-extractor`      | 提取 trace_id           | `log_data/trace_extractor.py`           |
| `post-filter`             | 后过滤和脱敏                | `log_data/post_filter.py`               |
| `log-evidence-mapper`     | 转成 EvidenceItem       | `log_data/evidence_mapper.py`           |


## 3. 强制调用链

```text
Skill Step
  -> builtin.log-query plugin
  -> MCP log.query_by_trace_id / log.query_by_template
  -> Log Provider Adapter
  -> ToolCallResult
  -> EvidenceItem
```

## 4. 禁止事项

- 禁止默认 Flow 直接调用 `SlsAdapter`。
- 禁止 Agent Runtime 直接构造 `LogQueryRequest`。
- 禁止未注册插件的临时日志查询工具进入生产。

## 5. 验收标准

- 任意日志查询都有 `skill_name`、`plugin_id`、`mcp_tool_name`。
- 任意查询结果都能变成 Evidence。
- 任意查询都能审计。
- 任意查询都能回放。
