# T18 回放与评估任务树

## 1. 目标

本任务把 `22-回放与评估基准.md` 落成可执行任务树，确保默认 Flow、Skill、MCP 工具、日志查询和根因分析都能被持续验证。

## 2. 模块拆分

| 模块 | 目标 | 建议文件 |
|---|---|---|
| `replay-case-contract` | 定义回放样本 | `contracts/replay.py` |
| `execution-trace-contract` | 定义执行轨迹 | `contracts/execution_trace.py` |
| `replay-store` | 存储回放样本和结果 | `replay/store.py` |
| `replay-runner` | 执行回放 | `replay/runner.py` |
| `metric-calculator` | 计算评估指标 | `evaluation/metrics.py` |
| `quality-gate` | 发布门禁 | `evaluation/quality_gate.py` |
| `evaluation-report` | 输出评估报告 | `evaluation/reporting.py` |
| `cli-replay` | CLI 回放命令 | `cli_commands/commands/replay.py` |
| `cron-replay` | 定时回放任务 | `cron/jobs/case_replay.py` |

## 3. 必备契约

### `CaseExecutionTrace`

必须记录：

- `case_id`
- `skill_name`
- `flow_plugin_id`
- `step_traces`
- `mcp_call_ids`
- `evidence_ids`
- `report_id`

### `SkillExecutionTrace`

必须记录：

- `skill_name`
- `skill_version`
- `step_name`
- `tool_calls`
- `inputs`
- `outputs`
- `errors`

## 4. 验收标准

- 默认 Flow 修改必须跑回放。
- Skill 发布必须引用评估结果。
- 回放样本必须脱敏。
- 评估结果必须能比较历史版本。
- 回放执行必须能追溯 Skill、Plugin、MCP、Evidence。
