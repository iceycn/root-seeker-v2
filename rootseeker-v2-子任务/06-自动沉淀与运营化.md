# T6 自动沉淀与运营化

## 1. 任务目标

本任务用于让 RootSeeker V2 不只是一个“执行排查”的系统，而是一个“会持续积累排查经验”的系统。

这一阶段的核心是把高质量 Case 执行过程转成可复用资产，并建立技能的版本治理能力。

边界说明：

- `T6` 接收 `T3` 已定义好的 Skill 格式、Registry 和 Composer。
- `T6` 只负责 Case 结束后的生成、审核、发布、版本、回滚、评估。
- `T6` 不重新定义 `SKILL.md` 基础格式。
- `T6` 不重新实现 Skill 选择与组合。

---

## 2. 范围

本任务覆盖：

- Generated Skill Draft
- Skill Review
- Skill Publish
- Skill Versioning
- Skill Rollback
- Evaluation / Replay

本任务不覆盖：

- 完整运营后台 UI
- 全量团队流程一次性接入

---

## 3. 输入

- `T3` 的 Skill 体系
- `T5` 的 `CaseReport`
- 已完成 Case 的执行轨迹
- `22-回放与评估基准.md` 中定义的评估集

---

## 4. 输出

必须产出：

- `GeneratedSkillDraft`
- Draft Review 规则
- Skill 发布规则
- Skill 版本化规则
- Replay / Evaluation 机制
- Skill 发布门禁

---

## 5. 一级拆解

### `T6.1` 生成技能草稿

从高质量 Case 中抽取可复用排查流程。

### `T6.2` 建立审核流

确保生成技能不会直接污染正式技能目录。

### `T6.3` 建立版本治理

确保技能变更可追踪、可灰度、可回滚。

### `T6.4` 建立 Replay / Evaluation

确保技能质量可被验证。

---

## 6. 二级拆解

## 6.1 `T6.1` 技能草稿生成

### 模块任务

- 从 CaseReport 提取结论
- 从执行轨迹提取步骤
- 生成 `SKILL.md` 草稿
- 存放到 `skills/generated/`

### 类文件任务

- `skill_system/synthesizer.py`
- `skill_system/draft_builder.py`
- `storage/skill_store.py`

## 6.2 `T6.2` 审核流

### 模块任务

- 草稿人工审核
- 草稿修订
- 审核通过后发布

### 类文件任务

- `skill_system/review.py`
- `skill_system/publisher.py`
- `contracts/review.py`

## 6.3 `T6.3` 版本治理

### 模块任务

- 技能版本号
- 技能灰度发布
- 技能回滚
- 技能废弃标记

### 类文件任务

- `skill_system/versioning.py`
- `skill_system/rollback.py`

## 6.4 `T6.4` Replay / Evaluation

### 模块任务

- Case 回放
- 技能效果评估
- 失败案例归档
- 技能质量评分

### 类文件任务

- `tests/replay/`
- `tests/evaluation/`
- `analysis/evaluation.py`

---

## 7. 推荐运营规则

建议采用三级目录：

- `builtin/`
- `generated/`
- `team/`

建议采用三级状态：

- `draft`
- `reviewed`
- `published`

建议采用版本变更规则：

- 结构变化升主版本
- 步骤变化升次版本
- 文案变化升补丁版本

---

## 8. 风险

- 如果自动沉淀没有审核机制，会把错误经验沉淀进去
- 如果没有回放与评估，技能数量越多质量越不可信
- 如果没有版本治理，正式技能会变成不可控资产

---

## 9. 验收标准

完成本任务时，应该满足：

- 已完成 Case 可以产出技能草稿
- 草稿不能直接进入正式目录
- 技能具备版本和发布状态
- 技能变更可以被回放验证
- 技能效果可以被评估
- 未通过回放评估的 Skill 不能发布

---

## 10. 推荐实施顺序

1. 先做 `GeneratedSkillDraft`
2. 再做审核流
3. 再做版本治理
4. 最后做 Replay / Evaluation
