# 服务目录解析子技能

## 目标

把归整后的服务名映射到 RootSeeker 服务目录，获得日志源、仓库映射、负责人和允许工具等上下文。

## 触发时机

- `incident.normalize` 完成后立即执行。
- 在查日志、查代码、发送通知前执行。

## 工具

- `catalog.resolve_service`
- `catalog.get_log_sources`

## 执行准则

- 使用归整后的 `service_name`、`tenant`、`environment`。
- 先解析服务目录，再获取日志源。
- 如果服务目录没有命中，继续使用原始服务名排查，但必须把目录缺失记录到证据。
- 不要因为目录缺失而停止日志和代码排查。

## 输出要求

- 服务目录条目。
- 日志源列表。
- 仓库映射是否存在。
- 目录缺失或字段不完整时的限制说明。
