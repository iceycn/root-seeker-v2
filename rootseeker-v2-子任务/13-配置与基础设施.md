# T13 配置与基础设施

## 1. 任务目标

本任务用于借鉴 OpenClaw 配置和基础设施分析，为 RootSeeker V2 建立企业级基础设施层。

这个模块不应被视为附属能力。企业级故障排查系统必须从一开始就具备：

- 配置 schema
- SecretRef
- 安全文件写入
- 网络请求守卫
- 执行审批
- Agent Events
- System Presence
- Logging / Redaction
- 状态迁移

## 5. 一级拆解

### `T13.1` 配置系统

建立基础配置、插件配置、通道配置的 schema 合并和校验。

### `T13.2` SecretRef

支持 env、file、exec 等来源的密钥引用。

### `T13.3` Infra 安全底座

提供安全文件、原子写入、网络请求防护和执行审批。

### `T13.4` Agent Events

提供 Agent 生命周期、工具、审批、错误事件总线。

### `T13.5` System Presence

跟踪系统节点、Worker、Gateway、Scheduler 的存在状态。

### `T13.6` Logging / Redaction

提供敏感信息脱敏、诊断日志和子系统日志。

## 7. 风险

- 如果配置系统不支持插件 schema 合并，插件化会受限
- 如果没有 SecretRef，企业密钥会散落到配置文件和环境里
- 如果日志不脱敏，排查系统本身会成为安全风险
- 如果没有 Agent Events，Gateway 和 UI 无法可靠展示执行过程

---

## 8. 验收标准

- 配置可以校验并输出明确错误
- SecretRef 可以解析并审计
- 文件写入具备原子性
- 网络请求具备基础防护
- Agent 事件可以订阅
- 日志能脱敏敏感信息
- System Presence 可以展示关键节点状态

---

## 9. 推荐实施顺序

1. 先做 Config Schema
2. 再做 SecretRef
3. 再做 Safe FS / Atomic JSON
4. 再做 Agent Events
5. 再做 Logging / Redaction
6. 最后做 System Presence 和状态迁移
