# 报告通知子技能

## 目标

把已收集证据整理成排查报告，并在报告状态明确后发送通知。

## 触发时机

- 所有可用证据步骤完成后。
- 如果中间工具不可用，仍应生成带限制说明的报告。

## 工具

- `notify.send`

## 报告准则

- 分别总结输入归整、服务目录、日志、trace、仓库索引、代码证据。
- 每个结论都必须能追溯到证据项。
- 未配置工具、空日志、trace 缺失、索引不可用、仓库为空应展示为限制条件。
- 不要把观测缺口写成业务根因。

## 通知准则

- 只在报告已生成后调用 `notify.send`。
- 通知内容应包含服务名、标题、根因摘要、证据数量。
- 通知目标未配置时记录跳过原因，不应让主流程失败。
