# Trace 链路子技能

## 目标

基于 trace id 获取分布式调用链，判断故障边界和上下游责任位置。

## 触发时机

- 输入归整或日志查询得到 trace id 后。
- trace id 缺失时不应伪造 trace，只记录观测缺口。

## 工具

- `trace.get_chain`

## 分析重点

- 当前服务是否有异常 span。
- 错误是否来自上游传入、当前服务处理、下游依赖、超时、重试或熔断。
- span 是否缺失、链路是否断裂、采样是否不足。

## 输出要求

- trace id。
- span 数量。
- 关键 span 摘要。
- 故障边界判断。
- trace 工具不可用时的限制说明。
