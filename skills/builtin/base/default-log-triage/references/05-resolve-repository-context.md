# 仓库与索引子技能

## 目标

确认代码检索前置条件，知道有哪些仓库可查、索引是否可用，以及代码证据是否可信。

## 触发时机

- trace 和日志线索收集后。
- 执行 `code.search` 前必须先检查索引状态。
- 服务目录没有仓库映射、索引不清楚、搜索无命中时需要列仓库。

## 工具

- `index.get_status`
- `repo.list`

## 执行准则

- 先调用 `index.get_status`，确认索引是否 ready。
- 再调用 `repo.list` 获取已注册仓库。
- 仓库为空、索引未配置、索引过期时，只能说明代码证据受限，不能直接推断业务根因。

## 输出要求

- 索引 ready 状态。
- 仓库数量和仓库摘要。
- 仓库映射缺失或索引不可用的限制说明。
